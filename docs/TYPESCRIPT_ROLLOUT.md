# Поэтапное внедрение TypeScript

Текущее состояние: проект на JavaScript, без `tsconfig`.

## Когда имеет смысл

- Долгая поддержка и несколько разработчиков.
- Нужны жёсткие контракты с RealWorld API и Vuex payload.

## Запреты по проектным правилам

- Не использовать `any`; при необходимости — `unknown` и сужение типов.

## Этапы

1. **Инфраструктура** — добавить `typescript`, `@vue/cli-plugin-typescript` или аналог для выбранной сборки; `tsconfig.json` с `strict: true` или поэтапным включением строгости.
2. **Граница API** — типизировать `src/api/*.js` → `.ts`: ответы DTO (Article, User, Profile), ошибки Axios.
3. **Vuex** — типизировать `state`, payload мутаций и аргументы actions в затрагиваемых модулях.
4. **Компоненты** — переводить `.vue` на `<script lang="ts">` по мере касания файлов; общие типы в `src/types/`.

## Совместимость с Vue 2

- Использовать `vue-class-component` / Options API с типами или готовиться к Vue 3 + `defineComponent` в рамках отдельного эпика.

## Критерий первого PR

- Один-два API-модуля + минимальные `shims-vue.d.ts`, без массового переименования файлов.
